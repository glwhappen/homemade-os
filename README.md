这本书出版到现在已经很长的时间了，所以我们的电脑已经换了好几代了，作者说的好多东西可能我们都没听过，更没有见过了，所以针对这个我做了下面的优化

1. 删除掉了我们根本用不到的!cons_9x.bat，这个文件是针对Windows95/98/Me的用户的，现在是不可能有这样的系统了，除非你喜欢古董，然后把!cons_nt.bat这个文件名改成了cmd.bat，如果你打开这个文件看一下的话，其实就是在当前的目录下打开了一个cmd控制台。
2. 删除掉了原有的install.bat,这个批处理的作用是刻录一个软盘，别说软盘了，很多人连这个光驱都没见过，所以直接删除
3. 我把系统要用的代码放到了project文件夹中，tolset工具和他在相同的目录，我感觉这样文件比较整洁。
4. 所有的代码都用git进行管理，你现在看到的可能是最终的代码，或者很多天后的，如果你想看之前的，请去commit的log中寻找，每次的提交都是书上的一个步骤。

在整个代码的编译和运行过程中我们都需要用到toset这个工具目录，里面包含虚拟机和作者写的一些程序（在30天自制操作系统这本书里面都会提到），因为比较大也不会修改，我直接把他作为子仓库管理了，在克隆的时候需要把子仓库也克隆下来，可以在克隆时加上--recursive参数。

我的笔记会放到project的readme中，包含当前文件的说明，同样你也可以回滚到之前，查看过去的文件说明。
